HTML

  • HTML is the framework of a website
  • The code is rendered from top to bottom, so order matters! for example, the CSS file is included in the metadata to load with the page where the JS file is at the end so visuals load first
  • All of these lists were made using HTML!
  • An element is defined by a start tag, some content, and an end tag
  • Elements can be nested inside of eachother, called nested elements
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • CSS and JS code files can be linked to the HTML file using the link and script tags respectively

CSS

  • CSS is the "paintjob" on a website
  • The color, borders, and shadows on this page are made with CSS
  • Inline CSS is coded directly in an element in the HTML file
  • Internal style sheets are inside a style element inside the head element in an HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
  • External CSS style sheets are coded in a separate file, making design changes easier. it can be linked with a "link" element in the HTML "head" element
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • The border goes between margin and padding
  • You are able to adjust assets like images, font sizes, colors, and more!

GIT

  • Git is an open source version control software that lets multiple people make separate changes to a web page at the same time
  • GitHub is a web-based interface that uses Git
  • A repository, or repo, is a folder in which all files and their version histories are stored
  • A Branch is a workspace in which you can make changes that won't affect the live site
  • Commit changes to save a record of a change made to a file within a repo
  • A Pull Request, or PR, is a way to ask for changes made in a branch to be merged into another branch that lets multiple people review the work
  • After a pull request is approved, it will MERGE from one branch to another
  • Issues are how work is tracked when using GIT through a progress
  • Make sure to consistently complete the Git workflow to keep track and save of progress
  • GIT Bash is the terminal on your local system that can navigate and execute tasks from a single place with commands
  • git status: shows current branch
  • git pull origin main: Pull the most recent up to date code file from the main branch to your local system
  • git add -A: Stage the code to be commited
  • git commit -m "your message": Commit added code to be ready to be pushed to GitHub. Add a message to know what you did
  • git push origin branch-name: Push commited code to GitHub for a pull request
  • git checkout -b branch-name: creates new branch and switches to it

JavaScript

  • Java reads and executes code from top to bottom. Order matters!
  • JS is a Object Oriented System. An object is an instance of a class and almsot every element is an object
  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • An 'If' statement can run through multiple outputs that will only run if the input matches the criteria for a certain part of the block code that follows
  • A 'For' loop is a repeating line of code that helps control flow of code and run through an array in a variable
  • A function is simply a set of instructions that tells the computer how to perform a certain task. Don't forget to call the function in the code for it to work!
